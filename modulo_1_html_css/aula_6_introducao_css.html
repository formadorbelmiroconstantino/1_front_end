Introdução ao CSS

O CSS (Cascading Style Sheets ou Folhas de Estilo em Cascata) é a linguagem utilizada para definir a aparência visual de uma página web estruturada em HTML. Enquanto o HTML cuida do conteúdo (textos, links e imagens), o CSS cuida do design (cores, fontes, espaçamentos e layouts).

1. Estrutura Básica (Sintaxe)

O CSS funciona através de regras que dizem ao navegador como um elemento deve ser exibido. Uma regra básica é composta por:

Seletor: Indica qual elemento HTML será estilizado (ex: h1, p, .minha-classe);

Propriedade: O aspecto que você quer mudar (ex: color, font-size);

Valor: A configuração específica da propriedade (ex: blue, 20px).

Exemplo:
h6 { color: blue; /* Propriedade color: Valor blue (cor azul)*/ font-size: 24px; }

3. Selectores Comuns

Para aplicar estilos de forma precisa, você pode usar diferentes tipos de seletores:
Tag: Seleciona todos os elementos de um tipo (ex: p para todos os parágrafos);
Classe (.): Seleciona elementos que possuem um atributo class. Pode ser usado em vários elementos;
ID (#): Seleciona um único elemento específico que possui um id exclusivo;
Universal (*): Aplica o estilo a todos os elementos da página.

4. Por que o nome "Cascata"?

O termo "cascata" refere-se à ordem de prioridade que o navegador segue quando existem regras conflitantes para o mesmo elemento. O navegador decide qual estilo aplicar com base na especificidade do seletor e na ordem em que as regras aparecem no código.

Para começar a praticar, você pode consultar o Guia Básico da MDN Web Docs ou o tutorial interactivo do W3Schools.

Qual é o melhor tipo de css a utilizar num projecto, inline, interno ou externo e por que?

O CSS externo é considerado a melhor prática e o método mais recomendado para quase todos os projetos web em 2025. Abaixo está o porquê de cada um ser usado e a razão da preferência pelo externo:

1. CSS Externo (O Melhor e Mais Usado)

É um arquivo separado (ex: estilo.css) conectado ao HTML via tag < link>.

Por que usar:

Manutenibilidade: Você altera o design de centenas de páginas editando apenas um arquivo;
Performance (Cache): O navegador baixa o arquivo CSS uma única vez e o armazena em cache, tornando o carregamento das outras páginas do site muito mais rápido;
Organização: Separa completamente o conteúdo (HTML) do design (CSS).

2. CSS Interno (Uso Pontual)

Escrito dentro da tag < style> no < head> do próprio HTML.

Quando usar: É útil para protótipos rápidos ou quando uma única página do site precisa de um estilo exclusivo que não será repetido em nenhum outro lugar;
Desvantagem: Aumenta o tamanho do arquivo HTML e dificulta a manutenção se o projeto crescer.

3. CSS Inline (Evite ao máximo)

Escrito diretamente dentro da tag HTML (ex: < h1 style="color: blue;">).

Quando usar: Apenas para correções rápidas de última hora, testes imediatos ou em e-mails marketing (onde o suporte a arquivos externos é limitado);
Desvantagem: Polui o código, tem a maior prioridade (sobrescrevendo outros estilos) e torna quase impossível manter um padrão visual em larga escala.
