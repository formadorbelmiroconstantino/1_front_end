Introdução ao Bootstrap

O Bootstrap é o framework front-end mais popular do mundo para o desenvolvimento de sites responsivos e focados em dispositivos móveis. Criado originalmente por desenvolvedores do Twitter, ele utiliza HTML, CSS e JavaScript para facilitar a criação de interfaces modernas e consistentes. Por isso é que primeiro vimos os submódulos HTML, CSS e JavaScript antes desse submódulo que estamos a ver, Bootstrap.

Objectivo

Padronizar e acelerar a criação de interfaces web, oferecendo um sistema de grid responsivo e componentes prontos (botões, menus, formulários).

Uso

Construir sites responsivos que funcionam bem em diferentes tamanhos de tela e dispositivos.

Principais Características

  • Sistema de Grid: Utiliza um sistema de colunas flexível (Flexbox) que adapta o layout automaticamente ao tamanho da tela do usuário [4, 5].
  • Componentes Pré-fabricados: Oferece uma vasta biblioteca de itens prontos, como botões, formulários, barras de navegação (navbars), modais e carrosséis.
  • Mobile-First: O framework é projectado para priorizar a visualização em dispositivos móveis, expandindo-se para telas maiores conforme necessário.
  • Personalização: É possível utilizar variáveis SASS para customizar cores, espaçamentos e estilos globais.

Como começar

A versão mais actual e estável é o Bootstrap 5, que removeu a dependência do jQuery, tornando os sites mais leves e rápidos. Para começar a usá-lo, você pode:

  1. Via CDN: Adicionar os links de CSS e JS diretamente no seu arquivo HTML através do site oficial getbootstrap.com.
  2. Via Gerenciador de Pacotes: Instalar em projectos profissionais usando o comando npm install bootstrap através do Node.js.

Site Ofical do Bootstrap

O site oficial do Bootstrap é o getbootstrap.com.

O que é um framework?

Um framework é uma estrutura pré-definida de código, ferramentas e directrizes que serve como um "esqueleto" para o desenvolvimento de software, facilitando e acelerando a criação de aplicações ao fornecer funcionalidades prontas e padronizadas para tarefas comuns, como segurança, gerenciamento de base de dados e interface, permitindo que desenvolvedores foquem na lógica específica do projecto.

Como funciona:

  • Base e estrutura: Define a arquitetura básica do projecto, organizando arquivos, pastas e a forma como o código deve ser estruturado.
  • Componentes reutilizáveis: Inclui bibliotecas e módulos prontos para funções genéricas (autenticação, manipulação de dados).
  • Padronização: Impõe boas práticas e padrões da indústria, garantindo qualidade, segurança e compatibilidade.
  • Abstracção de tarefas: Automatiza tarefas repetitivas, liberando o desenvolvedor para criar as funcionalidades únicas da aplicação.

Principais vantagens:

  • Agilidade: Reduz o tempo de desenvolvimento ao não precisar "reinventar a roda".
  • Eficiência e qualidade: Código testado e validado pela comunidade, diminuindo bugs e erros.
  • Manutenção: Facilita a manutenção e a escalabilidade do projecto.

Exemplos:

  • Desenvolvimento Web: Django (Python), Ruby on Rails (Ruby).
  • Machine Learning: TensorFlow, PyTorch.
  • Aplicações Móveis: Flutter, React Native.
