Breakpoints no Bootstrap

Breakpoints no Bootstrap são larguras de tela predefinidas (como 576px, 768px, 992px, etc.) que determinam quando o layout de um site responsivo deve mudar, adaptando-se a diferentes dispositivos (celulares, tablets, desktops) para garantir a melhor experiência do usuário; eles funcionam como "pontos de quebra" para aplicar estilos via Media Queries CSS.

Como funcionam?

  1. Pontos de Transição: São valores de min-width (largura mínima) ou max-width (largura máxima) que o Bootstrap usa para disparar mudanças no design.
  2. Design Mobile First: O Bootstrap é "mobile first", ou seja, os estilos básicos são para telas pequenas, e os breakpoints adicionam estilos para telas maiores, como tablets e desktops.
  3. Classes de Layout: Você usa classes específicas (como col-md-6, d-lg-none) que se aplicam a partir de um determinado breakpoint, dizendo como os elementos devem se comportar em cada tamanho de tela.

Pontos de Interrupção Disponíveis

O Bootstrap inclui seis pontos de interrupção padrão, às vezes chamados de níveis de grid/grade , para construção responsiva. Esses pontos de interrupção podem ser personalizados se você estiver usando nossos arquivos Sass de origem.

Cada ponto de quebra foi escolhido para acomodar confortavelmente containeres cujas larguras sejam múltiplos de 12. Os pontos de quebra também representam um subconjunto de tamanhos de dispositivos e dimensões de viewport comuns — eles não visam especificamente todos os casos de uso ou dispositivos. Media Queries

Como o Bootstrap foi desenvolvido com foco em dispositivos móveis, usamos algumas media queries para criar breakpoints adequados para nossos layouts e interfaces. Esses breakpoints são baseados principalmente nas larguras mínimas da viewport e nos permitem redimensionar os elementos conforme a viewport muda.

Min-width

O Bootstrap utiliza principalmente os seguintes intervalos de media queries — ou breakpoints — em nossos arquivos Sass de origem para nosso layout, sistema de grid e componentes.

  • // X-Small devices (portrait phones, less than 576px) // No media query for `xs` since this is the default in Bootstrap
  • // Small devices (landscape phones, 576px and up)
    • @media (min-width: 576px) { ... }
  • // Medium devices (tablets, 768px and up)
    • @media (min-width: 768px) { ... }
  • // Large devices (desktops, 992px and up)
    • @media (min-width: 992px) { ... }
  • // X-Large devices (large desktops, 1200px and up)
    • @media (min-width: 1200px) { ... }
  • // XX-Large devices (larger desktops, 1400px and up)
    • @media (min-width: 1400px) { ... }

Max-width

Ocasionalmente, usamos media queries que vão na direcção oposta (o tamanho de tela especificado ou menor):

  • // `sm` applies to x-small devices (portrait phones, less than 576px)
    • @media (max-width: 575.98px) { ... }
  • // `md` applies to small devices (landscape phones, less than 768px)
    • @media (max-width: 767.98px) { ... }
  • // `lg` applies to medium devices (tablets, less than 992px)
    • @media (max-width: 991.98px) { ... }
  • // `xl` applies to large devices (desktops, less than 1200px)
    • @media (max-width: 1199.98px) { ... }
  • // `xxl` applies to x-large devices (large desktops, less than 1400px) @media (max-width: 1399.98px) { ... }

Entre Pontos de Interrupção

As media queries podem abranger várias larguras de breakpoint:

// Apply styles starting from medium devices and up to extra large devices
@media (min-width: 768px) and (max-width: 1199.98px) { ... }

Exemplos de Breakpoints Comuns (Bootstrap 5):

  • sm (Small): 576px e acima (celulares em paisagem).
  • md (Medium): 768px e acima (tablets).
  • lg (Large): 992px e acima (desktops).
  • xl (Extra large): 1200px e acima (desktops grandes).
  • xxl (Extra extra large): 1400px e acima (desktops ainda maiores).

Em resumo: Breakpoints são essenciais para a responsividade, permitindo que seu site se ajuste suavemente a qualquer tela, usando os pontos de largura definidos pelo Bootstrap para controlar a grade e o layout.
