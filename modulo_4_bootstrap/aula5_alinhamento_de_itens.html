Alinhamento de Itens no Bootstrap

No Bootstrap, o alinhamento de itens é feito principalmente com classes de utilitários, usando Flexbox (d-flex) para alinhamento horizontal (justify-content-*) e vertical (align-items-*), ou classes de texto (.text-*) para alinhar texto dentro de elementos, além de classes de alinhamento vertical (.align-*) para itens inline e células de tabela, permitindo centralizar, alinhar ao início/fim e ajustar verticalmente de forma responsiva.

1. Alinhamento com Flexbox (Vertical e Horizontal)

Use d-flex no container e classes específicas no container para os itens:

  • flex-row (padrão): Itens em linha (horizontal).
  • flex-column: Itens em coluna (vertical).
  • justify-content-* (eixo principal): start, center, end, between, around, evenly.
  • align-items-* (eixo cruzado): start, center, end, baseline, stretch.

Exemplo

<div class="d-flex justify-content-center align-items-center" style="height: 200px;">
<div>Conteúdo Centralizado</div>
</div>

Resultado

Conteúdo Centralizado

2. Alinhamento de Texto

Classes para alinhar texto em blocos e componentes:

  • text-start: Alinha o texto à esquerda (padrão).
  • text-center: Centraliza o texto.
  • text-end: Alinha o texto à direita.
  • Responsivo: Adicione breakpoints (ex: text-md-center) para alinhar em telas médias e maiores.

Exemplo

<p class="text-center">Este texto é centralizado.</p>

Resultado

Este texto é centralizado.

3. Alinhamento Vertical (Itens Inline e Células)

Classes para alinhar elementos inline, inline-block, e células de tabela:

  • align-baseline
  • align-top
  • align-middle
  • align-bottom
  • align-text-top / align-text-bottom.

Exemplo

<span class="align-middle">Texto com alinhamento médio</span>

Resultado

Texto com alinhamento médio

4. Alinhamento Individual de Itens Flex (Align Self)

Use em um item dentro de um container flex para mudar o alinhamento dele no eixo cruzado:

  • align-self-*: start, end, center, baseline, stretch.

Exemplo

<div class="d-flex u">
<div>Item 1</div>
<div class="align-self-center">Item 2 (Centralizado)
</div>
<div>Item 3</div>
</div>

Resultado

Item 1
Item 2 (Centralizado)
Item 3

5. Centralizar Blocos (com Margem Auto)

Para centralizar um bloco (div, etc.) que tem largura definida, use margens automáticas:

  • mx-auto: Centraliza horizontalmente (margens esquerda e direita automáticas).
  • my-auto: Centraliza verticalmente (em containers flex/grid).

Exemplo

<div class="mx-auto u" style="width: 200px;">
Bloco centralizado horizontalmente
</div>

Resultado

Bloco centralizado horizontalmente

Explicação

Este código usa o sistema de grid do Bootstrap para criar uma estrutura de três colunas de largura igual, centralizadas em um contêiner, que se ajustam automaticamente ao tamanho da tela, mostrando "One of three columns" em cada uma. As classes .container (largura fixa ou fluida), .row (linha), .col (colunas automáticas) e text-center (texto centralizado) definem o layout responsivo e o alinhamento.

Aqui está a explicação detalhada de cada classe:

  • <div class="container text-center">:
    • container: Define uma largura máxima para o conteúdo e o centraliza horizontalmente na página. É a base necessária para o sistema de grid.
    • text-center: Uma classe utilitária que alinha todo o texto dentro deste container ao centro.
  • <div class="row align-items-start">:
    • row: Cria uma "linha" para envolver as colunas. Ela utiliza Flexbox para organizar os elementos filhos.
    • align-items-start: Alinha as colunas no topo da linha. Se as colunas tiverem alturas diferentes, todas ficarão alinhadas pela borda superior.
  • <div class="col">:
    • col: Esta classe indica uma coluna de largura automática. Como existem três divs com a classe .col dentro da mesma linha, o Bootstrap divide o espaço igualmente (33,33% para cada uma).

Explicação

Este código apresenta a maior flexibilidade de alinhamento até agora. A principal diferença é que o alinhamento vertical não é mais aplicado à linha (row), mas sim a cada coluna individualmente.

Aqui está a explicação detalhada:

1. Alinhamento Individual (align-self-*)

Em vez de todas as colunas seguirem a mesma regra, cada uma tem sua própria posição vertical:

  • align-self-start (na primeira coluna): Esta coluna específica fica alinhada ao topo da linha.
  • align-self-center (na segunda coluna): Esta coluna fica centralizada verticalmente no meio da linha.
  • align-self-end (na terceira coluna): Esta coluna fica alinhada à base (fundo) da linha.

Explicação

Este código demonstra as diferentes formas de distribuir colunas horizontalmente dentro de uma linha (row) no Bootstrap, utilizando as classes justify-content.

Em todos os exemplos, cada linha possui apenas duas colunas de tamanho fixo (col-4). Como o sistema de grid do Bootstrap vai até 12, as duas colunas somam 8, deixando um espaço vazio de 4 unidades que é distribuído de formas diferentes:

1. Alinhamento por Lado

  • justify-content-start: Alinha as colunas à esquerda (no início da linha). O espaço sobra todo à direita.
  • justify-content-center: Alinha as colunas ao centro. O espaço vazio é dividido igualmente à esquerda e à direita.
  • justify-content-end: Alinha as colunas à direita (no fim da linha). O espaço sobra todo à esquerda.

2. Distribuição Espaçada (Flexbox)

  • justify-content-around: Distribui o espaço ao redor das colunas. O espaço entre as colunas é o dobro do espaço entre as bordas e as colunas.
  • justify-content-between: Empurra as colunas para as extremidades. Todo o espaço sobra apenas entre as duas colunas, eliminando margens externas.
  • justify-content-evenly: Distribui o espaço de forma totalmente igual. O espaço entre as colunas e as bordas é exactamente o mesmo que o espaço entre elas.

Resumo das Classes

  • col-4: Define que cada coluna ocupa 4 de 12 espaços (33.3% da largura).
  • container text-center: Centraliza o layout na página e o texto dentro das colunas.

Este código é um guia visual perfeito para entender como o sistema de alinhamento horizontal (Justify) do Bootstrap 5 funciona.

Explicação

Este código demonstra o uso da técnica de flutuação (float) combinada com um Clearfix para fazer com que o texto envolva uma imagem, um layout muito comum em artigos e blogs/portais de notícias.

Abaixo está a explicação detalhada de cada componente:

1. A Classe clearfix

A classe .clearfix é um utilitário essencial quando se usa elementos flutuantes.

  • Problema: Quando você usa um float, o elemento é "removido" do fluxo normal da página, o que pode fazer com que o container pai (a div) "encolha" e não reconheça a altura da imagem.
  • Solução: O clearfix força o container a expandir sua altura para envolver completamente os elementos flutuantes dentro dele, evitando que o conteúdo abaixo (fora dessa div) quebre ou suba para lugares indesejados.

2. Posicionamento da Imagem

  • col-md-6: Define que, em dispositivos médios ou maiores, a imagem ocupará 50% da largura total (6 de 12 colunas).
  • float-md-start: Faz a imagem "flutuar" à esquerda em telas médias ou superiores, permitindo que o texto suba e preencha o lado direito.
  • mb-3: Adiciona uma margem na parte inferior (margin-bottom) para o texto não encostar na imagem por baixo.
  • ms-md-3: Adiciona uma margem à esquerda (margin-start) apenas em telas médias, útil para criar espaçamento se houvesse algo à esquerda.

3. Comportamento do Texto

Os parágrafos <p> não possuem classes de grid. Por causa do float-md-start na imagem, o texto flui naturalmente ao redor dela. Se o texto for longo o suficiente, ele começará ao lado da imagem e continuará abaixo dela assim que o espaço lateral terminar.

Resumo Visual

  • Em telas grandes: A imagem fica à esquerda ocupando metade da largura, e o texto a envolve pelo lado direito e por baixo.
  • Em telas pequenas (celulares): Como as classes possuem o prefixo -md-, o efeito de flutuação é desativado. A imagem ocupará a largura total (ou o padrão do navegador) e o texto aparecerá normalmente abaixo dela, garantindo a responsividade do layout.
