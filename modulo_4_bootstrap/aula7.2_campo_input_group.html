Formulários no Bootstrap - Campo Input Group

O Input Group (Grupo de Entrada) no Bootstrap é um componente utilizado para estender campos de formulário, permitindo adicionar ícones, textos, botões ou outros elementos visualmente conectados na frente ou atrás do campo de entrada (<input>).

Como Funciona

Para criar um grupo de entrada, envolve-se o campo e o elemento adicional em um container com a classe .input-group.

  • Complemento de Texto/Ícones: Utilize um elemento (geralmente ) com a classe .input-group-text dentro do container.
  • Posicionamento:
    • No Bootstrap 4, usa-se .input-group-prepend para adicionar antes e .input-group-append para adicionar depois do input.
    • No Bootstrap 5, basta colocar o elemento adicional directamente antes ou depois da tag <input> dentro da div .input-group (as classes prepend/append foram removidas para simplificação).

Exemplos de Uso

  1. Símbolos Monetários ou Usuários: Adicionar um "@" antes de um campo de usuário ou "Kz/AOA" antes de um valor numérico.
  2. Botões de Acção: Acoplar um botão de "Pesquisar" ou "Enviar" directamente ao lado do campo de texto.
  3. Checkboxes e Radios: Inserir uma caixa de selecção dentro do grupo para habilitar/desabilitar campos específicos.
  4. Tamanhos: É possível controlar o tamanho do grupo inteiro usando as classes .input-group-sm (pequeno) ou .input-group-lg (grande).

Diferença entre input-group e form-group

  • .form-group (ou .mb-3 no v5): Serve para o espaçamento vertical entre diferentes campos de um formulário (margens inferiores).
  • .input-group: Serve para a união visual horizontal de um input com outros elementos.

Múltiplas Inputs

Embora vários <input> sejam suportados visualmente, os estilos de validação estão disponíveis apenas para grupos de entrada com um único campo <input>.

Botões com Menus Suspensos

Os Input Group Buttons with Dropdowns no Bootstrap servem para agrupar funcionalidades de selecção de contexto ou acções múltiplas directamente vinculadas a um campo de entrada.

Eles são úteis principalmente para:

 
  • Selecção de Contexto/Categoria: Permitem que o utilizador escolha o tipo de dado que está inserindo. Por exemplo, em um campo de busca, o dropdown pode permitir filtrar entre "Produtos", "Categorias" ou "Marcas" antes da digitação.
  • Troca de Unidades ou Moedas: Em campos de valores numéricos, o botão dropdown pode servir para seleccionar a moeda (AOA\R\(,US\), €) ou a unidade de medida (kg, g, mg) associada ao número inserido no input.
  • Acções Rápidas Relacionadas: Permitem agrupar várias acções que podem ser realizadas com o valor digitado, como "Salvar", "Salvar e Enviar" ou "Salvar como Rascunho" em um único conjunto visual.
  • Economia de Espaço: Substituem múltiplos botões individuais ou campos de seleção separados por um único componente compacto e integrado ao campo de texto.

Para implementar, utiliza-se a estrutura padrão de um .input-group e insere-se um botão com a classe .dropdown-toggle e o atributo data-bs-toggle="dropdown" ao lado do <input>.
