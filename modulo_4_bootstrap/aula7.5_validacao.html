Formulários no Bootstrap - Validações

As validações de formulário no Bootstrap 5 utilizam majoritariamente as pseudo-classes do CSS3 (:invalid e :valid) e a API de validação do navegador Constraint Validation API.

1. Validação do Lado do Cliente (Browser)

Para aplicar o estilo visual do Bootstrap, você deve adicionar a classe .needs-validation ao <form> e o atributo novalidate para desativar as bolhas de erro padrão do navegador.

O feedback é exibido através das classes .valid-feedback ou .invalid-feedback.

2. O JavaScript Necessário

Para que as cores de erro apareçam apenas após o clique no botão de envio, o Bootstrap exige um pequeno script que adiciona a classe .was-validated ao formulário:

(function () {
'use strict'
// Busca todos os formulários aos quais queremos aplicar estilos de validação var forms = document.querySelectorAll('.needs-validation')
// Faz um loop neles e evita o envio
Array.prototype.slice.call(forms)
.forEach(function (form) {
form.addEventListener('submit', function (event) {
if (!form.checkValidity()) {
event.preventDefault()
event.stopPropagation()
}
form.classList.add('was-validated')
}, false)
})
})()

3. Validação do Lado do Servido

Se vai validar os dados no backend (PHP, Node.js, etc.), não use .was-validated. Em vez disso, adicione manualmente as classes .is-invalid ou .is-valid diretamente nos inputs:

  • .is-invalid: Torna a borda vermelha e exibe o texto de erro.
  • .is-valid: Torna a borda verde.
